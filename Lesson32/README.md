Route53 - 分流东京和新加坡区，多值应答路由
=====================================

## 知识点

* 使用 Route53 - 分流东京和新加坡区
* 使用多值应答路由策略
* 为Web服务配置健康检查

## 实战演习

+ 启动各区的服务器
+ 设置DNS健康检查
  - 东京区IP
  - 新加坡区IP
+ 设置多值应答路由 - global.deeplearnaws.ml
+ 停止东京区服务, 确认DNS解析结果
